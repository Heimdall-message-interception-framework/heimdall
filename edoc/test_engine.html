collect_state/1 *

collect_state(SUTModule::atom()) -> #prog_state{}

Collects the program state for a given SUT module. Program state includes everything that the MIL knows + the concrete and abstract state of the SUT. In order for the latter to be included, sut_module:abstract_state_mod() has to be defined.

explore/7

explore(TestEngine::pid(), SUTModule::atom(), Config::any(), MILInstructions::[#abstract_instruction{}], NumRuns::pos_integer(), Length::pos_integer(), Timeout::pos_integer() | infinity) -> [{pos_integer(), history()}]

SUTModule: Name of the SUTModule to be used.
Config: The test config represented as a map from atom to config value.
MILInstructions: MIL Instructions to be used as part of the exploration.
NumRuns: The number of test runs to be performed.
Length: The length (number of steps) for each test run.

returns: A list of the perfomed runs (see run() datatype).

Orders the test engine to explore a given SUTModule with a given Scheduler.

SUTModules are expected to implement the SUTModule behaviour.

store_run/3 *

store_run(Run::history(), Scheduler::atom(), Config::#{atom() => any()}) -> ok

Stores a test run on disk in an mnesia database.

In order for this to work, an mnesia node has to be running. Normally, this is done automatically when the test engine is started by calling mnesia_functions:setup_db() .
